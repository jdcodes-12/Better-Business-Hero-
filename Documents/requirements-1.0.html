Functional Requirements


Inventory Management Requirements

  1. Name: Create_InventorySheet

    Type: Functional

    Summary: Create a new live, unused inventory sheet.


  2. Name: Read_InventorySheet

    Type: Functional

    Summary: Choose and read a inventory sheet from the MS access database.


  3. Name: Update_InventorySheet

    Type: Functional

    Summary: Used with Read Inventory Sheet; Make an active inventory sheet, enabling adjustments.


  4. Name: Delete_InventorySheet

    Type: Functional

    Summary: Delete the active live inventory sheet and remove from database.


  5. Name: Import_InventorySheet

    Type: Functional

    Summary: Import data from other databse software (future increment - if time).


  6. Name: Export_InventorySheet

    Type: Functional

    Summary: Export active inventory sheet into .CSV format for future use.


  7. Name: View_InventorySheet

    Type: Functional

    Summary: Search from inventory sheet database and select a sheet to bring to active.

Non-functional Requirements

Constraints

  1. Name: Technical Aptitude

    Type: Constraint

    Summary: System users must have some skill with using computers and know how to navigate the system


  2. Name: Security Levels

    Type: Constraint

    Summary: System users will be limited on privileges based on security level.


  3. Name: Platform Requirements

    Type: Constraint

    Summary: Systems must run on Windows OS.


  4. Name: Platform Add-ins

    Type: Constraint

    Summary: As of Version 1.0, system only works with MS Access and MS Excel.


Risks

  1. Name: Check_Security_Level

    Type: Risk/Constraint

    Summary: Before system logins, security level should be recognized based on user credentials.


  2. Name: Check_Connection_To_Access

    Type: Risk

    Summary: Before Reading or Updating sheets, check that a database link exsists between the respective table.


  3. Name: Check_Access_File_Formatting

    Type: Risk/Constraint

    Summary: Check that files exported to .CSV files.


  4. Name: Check_Excel_Formatting

    Type: Risk/Constraint

    Summary: Check that excel formatting is correct on exporting active sheets; Create "dummy" sheet.


  5. Name: Check_Excel_File_Formatting

    Type: Risk/Constraint

    Summary: Check that files exported to .CSV files.


  6. Name: Check_Login_Info

    Type: Risk

    Summary: Check users EIN against Employee Database, looking for a match to access system.


  7. Name: Save_Login_Info

    Type: Risk/Constraint

    Summary: Check that each user EIN is unique and correctly saved to Employee Database.




Expected Changes

  1. Name: User_Interface

    Type: Functional/Constraint

    Summary: Being open to functional changes/non-functional changes that users need/want. Changes made on feedback after deployment


  2. Name: Modify Database

    Type: Functional

    Summary: May reorganize database tables for efficiency (i.e. moving the asset collection and liability collection into financial reporting).
